# Modernize UI5 freestyle JS app ‚ûú UI5 TypeScript app

Convert a legacy UI5 freestyle JavaScript app (typical 2018-2021 era ‚Äî `jQuery.sap.*`,
sync bootstrap, manual model creation, ES5, no tests) into a modern UI5 TypeScript app on the
latest 1.x release with proper async loading, manifest-driven configuration, BaseController
extension, and ui5-linter clean.

> **Status: stub.** This skill is a placeholder ‚Äî the full version comes after `migrate-segw-to-rap.md`
> is iterated. Below is the planned shape so you know what's coming.

## Planned phases

1. **Discovery** ‚Äî read the legacy app's `manifest.json`, `Component.js`, controllers, views, formatter; classify each finding as *blocker / cleanup / cosmetic*.
2. **Modern scaffolding** ‚Äî `mcp__SAPUI5_MCP_Server__create_ui5_app` to bootstrap `modern-ui5-app/` with TS template.
3. **Manifest mirror** ‚Äî translate legacy `manifest.json` to v1.60+ shape with `sap.app.dataSources` (no hardcoded URL), `sap.ui5.models` declarative, `sap.ui5.routing` proper.
4. **BaseController extraction** ‚Äî write `webapp/controller/BaseController.ts` with router/model/i18n helpers.
5. **Per-view conversion** ‚Äî translate each XML view + JS controller, replacing:
   - `jQuery.sap.declare/require` ‚Üí `sap.ui.define` (or ES module import in TS)
   - `var that = this; function(){ ... }` ‚Üí `const that = this; () => { ... }` or proper `bind`
   - Globals-formatter (`'window.com.demo...formatter.X'`) ‚Üí controller-relative `'.formatter.X'` with required module
   - `oCtx.getPath()` regex parsing ‚Üí `oCtx.getProperty("Key")`
6. **TS-specific rewrites** ‚Äî types from `sap-ui5-types`, `import` statements, no implicit `any`.
7. **Linter pass** ‚Äî `mcp__SAPUI5_MCP_Server__run_ui5_linter` until clean.
8. **Manifest validation** ‚Äî `mcp__SAPUI5_MCP_Server__run_manifest_validation` until clean.
9. **Smoke test** ‚Äî `npm install && npm start` in `modern-ui5-app/`, verify against the same legacy V2 OData service (or the new V4 service from `migrate-segw-to-rap`).
